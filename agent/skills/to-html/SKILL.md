# /to-html — 最短理解インフォグラフィック

読み手がHTMLを読むだけで対象を**最短理解**——最短の時間で、過不足なく理解——できるようにする。以下のすべての判断はこの一点に従う。

## 1. 素材の確定

引数がファイルパス/URLなら全文を読む。トピック(依頼文)なら自分の知識から素材を組み立て、確信のない事実は調べて裏を取る。

次に**概念リスト**を作る: 解説対象を構成する概念を漏れなく列挙し、各概念に「載せる/落とす」を判定する。落とす基準——それを落としても読み手の理解・判断・行動が変わらないなら落とす。

完了条件: 全概念に載せる/落とすの判定が付いた概念リストがある。

## 2. 構成設計

1ページ=読み手の1つの問いに答える単位。答えるべき問いが複数あれば複数ページに分割し、index.html には読み手が最初に持つ問い(これは何か・なぜ必要か)を割り当てる。分割時は全ページ共通のナビゲーション(現在地表示付き)を置く。

完了条件: 「載せる」全概念に担当ページとセクションが割り当てられている。

## 3. HTML生成

出力先: このセッションを起動したディレクトリの直下に `to-html/<slug>/` を作る(slug は対象名の短い英語)。単一ページでも index.html、分割時は各ページ + 共有 style.css。

- 文章より図。関係は矢印図、対比は並置、手順は番号付きフロー、量の差は棒や面で描く。段落は図で表せない「なぜ」にだけ使う。
- 図はインラインSVGまたはHTML/CSSで描く。外部依存ゼロ(CDN・Webフォント・外部画像・JSライブラリなし)。ディレクトリごとコピーすれば単体で表示できること。
- 各ページは冒頭に全体像、以降は上から下への一本道。後のセクションを前提にした前方参照を作らない。
- 装飾は情報を運ぶものだけ。色は意味(状態・分類)と対応させ、意味のない色を増やさない。

## 4. 用語表

全ページを走査し、使われている各単語に**初心者テスト**をかける: その分野の初心者(IT系の解説ならジュニアエンジニア)がこの単語を知っていそうか? 知らなそうな単語をすべて2列の表(単語/意味)に載せる。意味は初心者テストに通る言葉だけで1〜2文。

置き場所: 複数ページなら glossary.html を作り全ページのナビゲーションから届くように。単一ページなら末尾セクション。

完了条件: 初心者テストに落ちる単語で、表にないものがページ上に1つもない。

## 5. 過不足検査

生成物を概念リストと突き合わせる。

- 不足: 「載せる」概念でページに現れないものはないか
- 過剰: ページ上の説明で概念リストに遡れないものはないか

引っかかったら該当ステップへ戻って直す。通ったら index.html の絶対パスを報告して終了。
